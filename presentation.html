1. We Started with the Data

Before an AI can learn, it needs data. I created two different kinds for this project.

üìù **Labeled Emails:** I generated a set of 100 sample emails in a JSON file. Each email was labeled with its correct `intent` and `urgency`. This is the data our first AI model would train on.

üìö **The Knowledge Base:** I also created a folder of internal documents as `.txt` files, such as project briefs and company policies. This acts as a private reference library for the AI to find facts.

2. We Built the "Fast Sorter"

The first part of the system needed to be very fast to handle the initial sorting of every email.

For this, I used a Machine Learning algorithm called **DistilBERT**. I chose it because it‚Äôs great at understanding the context of human language, which makes it very accurate.

I **trained** this model on the 100 sample emails. After studying these examples, the model learned the patterns and can now quickly predict the intent and urgency of any new email.

3. We Organized the "Library"

Just having a library isn't enough; the AI needs a fast way to find the right information. This is the core of the RAG system.

To do this, I used a **Vector Database** called **ChromaDB**.

The system reads all the documents, breaks them into small chunks, and converts each chunk's meaning into a number code, or a **vector**.

This allows the AI to search by **meaning**, not just keywords, and instantly find the most relevant information.

4. We Added the "Expert Brain"

The fast sorter is great, but for deep thinking and writing, we need a more powerful brain.

üß† **The "Expert Brain":** I used a powerful Large Language Model called **Mistral-7B** that runs entirely on my own computer. This makes the system completely private.

üîó **The "Glue":** To make all these different parts work together, I used a framework called **LangChain**. It acts as the central nervous system, passing the email from one stage to the next.

5. We Built the "Front Door"

With the whole system built, we needed a way for the outside world to use it.

I used **FastAPI** to build a web API. This acts as the front door to our application.

It creates a simple web address where we can send an email and get the full analysis back, turning the project from a script into a professional, usable application.
